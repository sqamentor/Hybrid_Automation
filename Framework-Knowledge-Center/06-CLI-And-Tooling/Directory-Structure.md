## Maintenance Guidelines

### Adding New Projects

1. Add project to `config/projects.yaml`
2. Create: `pages/<project>/`, `tests/modern/<project>/`, `recorded_tests/<project>/`, `test_data/<project>/`
3. Create `conftest.py` fixtures for the project
4. Update `Framework-Knowledge-Center/` documentation

### Adding New Scripts

1. Determine category: `setup` / `validation` / `audit` / `utilities`
2. Place in `scripts/<category>/`
3. Name clearly: `verb_noun.py`
4. Add docstring and usage instructions

### Organizing Artifacts

1. All outputs go to `artifacts/`
2. Use dynamic naming with timestamps
3. Configure `.gitignore` to exclude content
